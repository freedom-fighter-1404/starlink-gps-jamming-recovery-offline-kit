# نکات امنیتی (برای محیط‌های پرریسک)

این کیت برای اشتراک‌گذاری آفلاین (USB، AirDrop، انتقال فایل محلی) طراحی شده است. در محیط‌های پرریسک، فرض کنید که فایل‌ها ممکن است **در مسیر توزیع دستکاری شوند**.

## چه چیزهایی را صحت‌سنجی کنیم؟

### ۱) صحت‌سنجی پوشه بعد از Extract (توصیه‌شده)
از داخل پوشه کیت:
- Windows: `verify_integrity.bat` (یا `verify_integrity.ps1`) را اجرا کنید
- macOS/Linux: `./verify_integrity.sh` را اجرا کنید

### ۲) بررسی سریع چشمی (Sanity Check)
برای فایل‌های کاربرپسند (`START_*` و `scripts/starlink_gps.sh`) این موارد را چک کنید:
- **IP دیش:** `192.168.100.1`
- **پورت:** `9200`
- **متد:** `SpaceX.API.Device.Device/Handle`
- **درخواست:** `dish_inhibit_gps`

اگر IP/دامنه‌های دیگری دیدید یا چیزی شبیه ردیابی/ارسال اطلاعات بود، **کیت را اجرا نکنید**.

## مدل تهدید (واقع‌بینانه)
این صحت‌سنجی کمک می‌کند این موارد را تشخیص دهید:
- تغییر فایل‌ها برای اتصال به سرورهای غیرمحلی
- اضافه شدن کدهای ردیابی/Telemetry
- خرابکاری برای اتلاف وقت کاربران

اما از این موارد محافظت نمی‌کند:
- آلوده بودن دستگاه (بدافزار)
- دسترسی فیزیکی به دستگاه
- آلوده بودن باینری `grpcurl` از یک ساخت نامطمئن (طبق `docs/VERIFY_FA.md` باینری‌های باندل‌شده را هم بررسی کنید)

## اگر این کیت را توزیع می‌کنید
- لینک رسمی انتشار و هش SHA‑256 فایل ZIP را از **حداقل دو مسیر مستقل** منتشر کنید.
- اگر از امضا استفاده می‌کنید، اثر انگشت کلید عمومی (`release_keys/FINGERPRINT.txt`) را هم از مسیرهای مستقل منتشر/تأیید کنید.
- به کاربران توصیه کنید اول هش ZIP را بررسی کنند، سپس صحت‌سنجی داخلی کیت را اجرا کنند.
